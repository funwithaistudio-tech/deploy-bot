## ✅ 1. **Code Correctness and Potential Bugs**

### 🔍 Issues Identified:

#### **A. Incomplete GitHub API Usage**
- The `GhApi` client is initialized without an access token, which will likely cause **unauthenticated API calls** and hit rate limits or fail to access private repositories.
  - **Fix**: Pass the GitHub token via `token=` parameter in `GhApi(...)`.
  - Example:
    ```python
    GhApi(owner=owner, repo=repo, token=os.environ.get("GITHUB_TOKEN"))
    ```

#### **B. Incorrect Branch Logic for Preview Deployments**
- The logic:
  ```python
  is_preview = args.branch != "main" and not args.branch.startswith("preview/")
  ```
  is **flawed**. It assumes that any branch that is not `main` and not starting with `preview/` is a preview deployment. This is ambiguous and may not reflect actual intent.
  - **Fix**: Clarify and simplify the logic. For example:
    ```python
    is_preview = args.branch != "main"
    ```

#### **C. No Handling of Workflow Run Status**
- The code assumes that if a workflow exists, it was successful. It does **not check** the `conclusion` field.
  - **Fix**: Add a check:
    ```python
    if workflow_status.get("conclusion") != "success":
        print("⚠️ Workflow did not succeed. Aborting deployment.")
        return
    ```

#### **D. No Error Handling for Missing Vercel Token**
- The code warns about missing token but still proceeds with a "demo" deployment. This may be misleading.
  - **Suggestion**: If deployment is critical, exit early if no token is provided.
